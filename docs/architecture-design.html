PLRA Blockchain Land Records System

Clear Architecture Design Diagram

This design uses a hybrid layered architecture. Citizens and government officers use React portals, the business logic runs inside one modular Node.js/Express backend, PostgreSQL stores operational data, and Hyperledger Fabric stores trusted registration and transfer ledger records.

Frontend Portals Modular Backend PostgreSQL Database Hyperledger Fabric Monitoring + Recovery

Why This Design

Separate user roles make the workflow clear.
Frontend portals give role-based dashboards.
Core modules handle registration, transfer, and blockchain.
Supporting modules add succession, monitoring, analytics, and controls.
PostgreSQL keeps fast relational and workflow data.
Fabric keeps trusted ledger verification for critical actions.

5. What Happens in the System

Workflow View
1

User starts action

Citizen, LRO, DC, or Admin opens the frontend portal and performs an action like registration, transfer, succession, or monitoring.

2

Backend applies business rules

The Node.js backend checks role permissions, validates the workflow, updates the right module, and records the new state in PostgreSQL.

3

Trusted record is committed

Critical registration and transfer outcomes are pushed to Hyperledger Fabric for trusted ledger proof and later verification.

Design Choice: This is a hybrid layered architecture with a modular monolith backend. It fits your project because one backend coordinates many land-record workflows, while PostgreSQL handles application data and Hyperledger Fabric handles trusted blockchain proof.
