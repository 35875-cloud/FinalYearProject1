5. Blockchain Network Deployment

Fabric Nodes
GW

Fabric Integration Node

Fabric Gateway / Client Layer

The backend connects to the Fabric network through configured gateway services and connection profiles.

  • Channel: landregistry
  • Registration chaincode: voting
  • Transfer chaincode: land-agreement
FN

Hyperledger Fabric Network

Peer Nodes

Endorse and query blockchain transactions for registration and transfer workflows.

Orderer Nodes

Provide Raft ordering service and channel block commitment.

Ledger

Stores committed registration and transfer proofs.

Docker Containers

Peers, orderers, and chaincode runtime are deployed as containerized services in the Fabric environment.

6. Communication Paths

Interaction Paths

Client Browser → Frontend → Backend API

Users interact through the browser. The frontend sends REST requests to the Node.js backend for registration, transfer, succession, analytics, and restrictions.

Frontend ↔ Socket.IO Runtime

Negotiation chat and selected updates are delivered through realtime WebSocket communication.

Backend → PostgreSQL

The backend stores and reads operational workflow data from the relational database.

Backend → Fabric Gateway → Hyperledger Fabric Network

Approved registration and transfer operations are committed to the blockchain through Fabric gateway integration and chaincode invocation.

Ops Services → Backup Storage / Monitoring / Recovery

Audit, backup, alerting, integrity verification, and admin recovery features support operational safety and trust.
